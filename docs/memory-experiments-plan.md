## 2. Shared statistical protocol (referenced by every closed-loop experiment)

- **Pairing is matched-randomization, not bitwise identity.** The harness fixes all randomization: `np.random.seed(7)` (`eval_libero.py:94`), `env.seed` (`:303-305`), fixed init state `initial_states[episode_idx]` (`:153`), per-episode flow-noise `torch.Generator` seeded by the monotonic episode counter (`model2libero_interface.py:70-76` → `websocket_policy_server.py:151-153` → `VLA_JEPA.py:576-580`); no mode consumes extra RNG draws. However, closed-loop rollouts are **not reproducible run-to-run**: two identical allv2 repeats differ by 6 pp on libero_10 (`SLURM/vlajepa_evalall_601583{4,6563}.out`) — GPU nondeterminism plus chaotic divergence after the first differing action. Therefore the pairing claim is exactly this: episode k shares (task, init state, noise seed) across modes; trajectories coincide only until actions diverge. McNemar remains valid under matched randomization; T0.0's ICC from k≥5 repeats quantifies how much variance the pairing actually removes.
- **Known noise:** ±6 pp/suite at 100 episodes. Provisional design: libero_10 at **50 trials/task** for headline modes (all 50 init states verified in `init_files/libero_10/*.pruned_init`), **30 trials/task** for T0.3/T0.4 manipulation arms (libero_10 only), 10 trials/task for exploratory suite coverage; headline triad additionally at eval seeds {7,17,27} on libero_10. **All Ns confirmed or escalated after T0.0 calibration, before unblinding.**
- **Tests:** McNemar exact on discordant pairs (co-primary endpoints, Holm), stratified per-task paired bootstrap 95% CI, Holm correction across secondary modes. Offline: Wilcoxon signed-rank + episode-level bootstrap on ≥2,000 paired segments.
- **Records:** every episode → JSONL (schema in §7 H6); no result is ever derived from `eval.log` greps.

### T0.4 — Content specificity: memory transplantation and corruption (closed loop)
- **Hypothesis:** the policy reads episode-specific content: live > cross-task-donor ≥ noise-matched ≈ bypass; slot permutation and cross-episode carryover degrade toward bypass. If foreign ≈ live ≈ noise-matched, the fusion path responds only to activation statistics and the "memory" interpretation is falsified.
- **Method (trimmed to 4 decisive arms per power review; libero_10 only, 30 trials/task):** donor bank from T0.1 live dumps (no new recording pass). Server modes (§7 H5): **`foreign`** (cross-task donor state loaded at reset from MEMORY_DONOR_DIR, matched donor decision index to control state maturity, injected via the existing `memory_state` kwarg, `VLA_JEPA.py:529-533`; `update_memory=False` → persistent wrong content — the decisive version); **`noisematch`** (Gaussian donor with per-slot-channel moments fitted to the state bank — separates wrong content from off-manifold magnitude; reported as the statistics control, with on-manifold foreign donors as the honest primary per reviewer:E3); **`permute`** (permute the 8 working slots before injection — destroys slot_ids binding, preserves content; replaces the vacuous-at-B=1 `shuffle_within_batch`, stated in the paper); **`noreset`** (skip clearing state in the reset handler, `websocket_policy_server.py:147-166` — episode-isolation check). **Cut from Tier 0:** `foreign_sametask` (episode-vs-task specificity — Tier-2 filler if the decisive arms show content specificity) and the transient (`update_memory=True`, washout) variant — the latter is a cheap add-on ONLY if G1 is positive.
- **Data:** libero_10; donor bank ≈ 650 KB/episode. **Controls:** live and bypass anchors reused from T0.1, paired on the shared first-30 init states; ordering pre-registered; matched donor decision index.
- **Metrics:** paired success per condition vs live (McNemar + CI); isotonic ordering test live > cross-task ≥ noise-matched; noreset − live (reset-independence success criterion, Design-Proposal.md:320-332).
- **Logging/viz:** episodes.jsonl with {donor_source, donor_task, donor_decision, write_policy}; per-decision working-norm/injection telemetry. Figure: single "state manipulation" bar chart (6 conditions incl. anchors, CI whiskers, live highlighted).
- **Cost:** 4 arms × 300 eps ≈ **12 GPU-h.** — **Feeds:** Fig. "Memory transplantation" — the placebo-arm structure memory-paper reviewers expect.

### T0.5 — Observation-blackout stress test: does memory bridge dropouts? *(flagship behavioral figure)*
- **Hypothesis:** inference consumes only current frames (`VLA_JEPA.py:544-549`), so during a D-decision blackout memory is the only history channel: live degrades less than bypass, the gap grows with D, and shrinks toward the live-vs-**prior** null when the blackout covers decision 0 (memory holds only the trained prior) — a built-in placement falsification. Standard LIBERO is largely Markovian; this manufactures the memory demand from existing assets.
- **Method:** client-side corruption aligned to decision boundaries (server called when step % 7 == 0, `model2libero_interface.py:123-132`): corrupt env steps [7·d0, 7·(d0+D)) after the rotation at `eval_libero.py:176-179`. Args: d0 ∈ {0 (placement control), 8, 16 (libero_10)}, D ∈ {1,2,4}, fill ∈ {freeze_last_frame, black}, views ∈ {agentview, both}. **Placement-control fix (per review):** at d0=0 live still injects the trained prior, so live−bypass need not vanish; the d0=0 cells therefore compare **live vs prior** (correct empty-memory reference), with bypass retained as a tertiary line. Sub-ablation: per-request `suppress_write` payload field honored at `websocket_policy_server.py:209-214` — live-writing-during-blackout vs write-masked (does the learned update gate already reject corrupt evidence?). Core modes {live, bypass} under identical schedules; {prior} added at d0=0.
- **Data:** libero_10 + libero_goal; trimmed core grid (black fill, both views) first, freeze fill second. **Controls:** D=0 arms are exactly T0.1 (anchors); d0=0 placement control (live vs prior); identical schedule across modes (matched randomization); freeze-vs-black separates "detects corruption" from "lacks information".
- **Metrics:** success vs D per mode (paired CIs); interaction test (logistic regression, mode×D term); write-masked vs write-live delta; blackout-survival rate.
- **Logging/viz:** episodes.jsonl + {blackout_start, D, fill, views, suppress_write}; corrupted-frame strips saved per condition; rollout MP4s kept for qualitative panels. Figure: success vs D lines (live, live-write-masked, bypass) with CI bands, one panel per fill; inset blackout frame strip; d0=0 live-vs-prior collapse panel.

### T0.6 — Probe, geometry, and slot-anatomy suite: what the memory encodes and how
- **Hypothesis:** the 8×512 working state linearly decodes episode variables (task ID, phase quintile, progress, gripper-event count) above present-only-token, shuffled, constant-initial-slots, and untrained-module controls; marginal decodability of lagged robot state decays consistently with the gate-predicted (1−ḡ)^D retention; states trace a smooth task-clustered progress manifold; slots are non-degenerate (pairwise cos < 0.95) with heterogeneous causal knock-out effects.
- **Method:** **states, diagnostics, and knock-outs come from the T0.2 replay engine with H3 capture enabled (one replay engine, two analysis frontends — the standalone `replay_memory_states.py` is cut per review).** Per decision dump: working state [8,512] fp32, mean-pooled action/embodied tokens, labels (task, episode, decision idx, t/T, gripper channel), write-attention maps, update-gate stats, per-slot Δ norms. Analyses (CPU unless noted): **(a)** logistic/ridge/MLP probes with 5 episode-level splits; **(b)** lagged-state retention: ridge on `observation.state` [8] read directly from LeRobot parquets at frame t−7D (model never consumes it — `with_state: false`, `datasets.py:1882-1887` — clean external label), lags D ∈ {0,1,2,4,8,16}; marginal ΔR² = R²(present+memory) − R²(present); exponential fit vs measured-gate prediction; DROID extension for long lags (~2 GPU-h; lag 16 = 112 frames, well within DROID episode lengths); **(c)** geometry: global PCA, per-episode trajectories colored by t/T, Spearman ρ(progress direction, t/T), task silhouette in memory vs token space, effective rank (reuse estimators `jepa_analysis.py:54-74`); references = present-token geometry, prior-mode replay, untrained-module replay (`_reset_parameters`); **(d)** slot anatomy: pairwise slot cosine over decisions; fusion read-attention capture (`need_weights=True` behind flag, `fusion.py:91-96`); per-slot knock-out — zero `memory_read.tokens[:, i, :]` before fusion with identical `initial_noise`. **Honesty fix (per review):** single-slot zeroing is off-distribution — `recurrent_memory.py:221-223` zeros whole rows (all 8 slots), never one slot among seven live ones; we drop that justification and rely on the controls: no-op noise floor, all-slots-zeroed max reference, matched-norm random replacement, permutation-null heterogeneity test; **(e)** write microscopy: per-slot gate heatmaps, write-attention entropy vs ln(24) with untrained-module null band.
- **Data:** all 4 LIBERO suites (1,693 eps, ~39K decisions); enumerator-verified unseen DROID episodes for lags. **Controls:** as enumerated per analysis; all splits episode-disjoint, fixed across feature sets (paired at fold level).
- **Metrics:** balanced accuracy / R² with 95% bootstrap CIs (1,000 resamples, 5 splits); ΔR²(D) decay constant ± CI vs (1−ḡ); ρ median+IQR; silhouette; knock-out ‖Δaction‖ + Δ open-loop MSE per slot with CIs.
- **Logging/viz:** npz shards + parquet index under `results/probes/step_34729/`; figures: probe bar chart (target × representation), retention curve with gate-predicted overlay, PCA manifold with task hulls, slot-cosine heatmap + read-attention map + knock-out bars, gate/entropy microscopy panels.
- **Cost:** replay ~3 GPU-h (shared with T0.2 where lattices overlap), knockout ~4, DROID ~2 → **~9 GPU-h**; probes CPU. — **Feeds:** Fig. "What memory stores", Fig. "Retention horizon", Fig. "Memory manifold", Fig. "Slot anatomy" (appendix).

### T0.7 — Checkpoint archaeology: head-vs-memory decomposition across training *(sparse anchors)*
- **Hypothesis:** the cotrain action-loss drop (0.162→0.132) decomposes into a head effect (bypass-mode offline error improving across checkpoints) and a memory effect (the paired bypass−live gap widening as the gate grows 0.0125→0.0219). If the gap stays ~0 at every checkpoint, the memory contributes nothing during training and gap #1 stands.
- **Method:** rerun **only the T0.2 replay stage** (Qwen caches bit-identical across checkpoints — frozen since stage1) plus a reduced T0.6 battery (task-ID probe, lag-4 ΔR², gate dispersion, write entropy on a fixed 300-episode subset) on the **surviving anchors only: stage1 step_5000, cotrain 27752/30000/34729**, plus every archived 10K save from T0.0 onward through 100K. **Stated limitation (per review):** the 5K→27.7K window — where most gate growth occurred — was destroyed by `keep_last_checkpoints: 3` before this plan existed; the figure is drawn as sparse anchor points with the gap explicitly marked, no interpolation, and the caption says so. Extract tanh(gate) per checkpoint from safetensors (CPU); injection ratio per checkpoint from the replay activations. Fixed probe subset, splits, and noise across all checkpoints.
- **Data:** fixed 300 LIBERO + 100 enumerator-verified DROID episode subset. **Controls:** stage1 step_5000 = pre-cotrain anchor; untrained-module floor at every point; shuffled-state gap per checkpoint verifies the widening gap is content-driven, not gate-magnitude-driven; frozen protocol → differences attributable to weights only.
- **Metrics:** bypass MSE(step); paired gap(step) with CIs; memory-attributable fraction = gap(step)/[bypassMSE(stage1) − liveMSE(step)]; probe/entropy trajectories; Spearman correlation with `loss/vla_action_loss` from TB event files (loss curves survive the pruning even where checkpoints don't).
- **Logging/viz:** checkpoint × metric CSV under `results/probes/archaeology/`; master multi-panel figure: bypass error, paired memory gap, gate value, probe accuracy, write entropy vs training step — **sparse-anchor rendering with the destroyed window shaded**; snapshot manifest (SHA/paths).
- **Cost:** ~1 GPU-h/ckpt replay + ~2 GPU-h/ckpt reduced probes × ~10 ckpts ≈ **15 GPU-h** (spread over training; final checkpoints as they land). — **Feeds:** Fig. "Emergence of memory use" — rebuttal to "your loss improvement is just the trainable head", complementing T1.1.

### T0.8 — Versioned memory-dependent LIBERO variant: occluded-cue pilot *(new; covers the strongest open reviewer objection)*
- **Hypothesis:** on a task where the information needed at decision d > k is visible only at decisions < k, live outperforms prior and bypass; on the unoccluded twins the three modes match. This is the memory-required-by-design benchmark that Evaluation-Plan.md:127-138 demands and that standard LIBERO + inference-time blackouts (T0.5) do not provide.
- **Method:** build `libero_mem_v0` (versioned, shipped as an artifact regardless of outcome): 3 tasks derived from libero_object/spatial pick-place BDDLs. Variant (a) **occluded-target:** after k ∈ {2,4} decisions, insert/move an occluder geom between agentview camera and the target region via seeded mujoco `sim.data.qpos` edit + `sim.forward()` at a decision boundary (same scene-edit helper as T2.1, §7 H6); target remains physically reachable but invisible in agentview; wrist view untouched. Variant (b) **randomized-start control:** target position randomized across episodes at t=0 (init-state edit) so a present-frame or task-prior shortcut cannot substitute for memory of the observed location. Guard `env.check_success()` semantics and `done` behavior; **1-task smoke run before committing the grid** (shared requirement with T2.1). Modes {live, prior, bypass} × {occluded, unoccluded twin} × 3 tasks × 30 trials.
- **Data:** libero_object/spatial assets + edited init files, versioned under `benchmarks/libero_mem_v0/`. **Controls:** unoccluded twins (mode differences must match T0.1); occlusion-at-d0=0 arm (target never seen → all modes should fail equally, placement falsification); identical edit schedule across modes.
- **Metrics:** paired success live vs prior (primary for this experiment), live vs bypass; interaction (occluded × mode); success vs k.
- **Logging/viz:** episodes.jsonl + {occluder_onset_k, variant}; frame strips of occlusion onset; **Fig. "Memory-dependent benchmark (libero_mem_v0)"** — mode bars, occluded vs unoccluded panels.
- **Stated limitation and mitigation:** the model was never *trained* under occlusion, and the training pipeline (4-decision credit window, detached burn-in, gate 0.022) was never given a strong incentive to store task-critical content — a null here is diagnostic of the training design, not only of the module. Mitigation: T2.3 (context/credit-horizon branches) is the paired redesign evidence; the benchmark itself is a contribution and seeds the next training iteration (train-under-occlusion is future work, scoped out).
- **Cost:** ~10 GPU-h + ~150 LOC shared with T2.1's scene-edit helper. — **Feeds:** Fig. "Memory-dependent benchmark"; the Level-C claim ladder.

## 4. Tier 1 — non-negotiable retrain baselines (~1,000–1,100 GPU-h staged)

### T1.1 — True no-memory co-train twin (same seed, same data order, matched steps) *(the attribution retrain)*
- **Hypothesis:** at matched step and identical data order, memv1 outperforms a bitwise-comparable no-memory twin on paired libero_10 success and matched-step action loss. If the twin matches, the action-loss improvement is fully explained by the co-trained head + segment-mode data (gap #1) and the contribution claim is falsified — automatic-rejection territory if this row is missing.
- **Method:** relax the guard at `VLA_JEPA.py:430-431` so `forward_sequence` runs memory-disabled (all memory ops already gated on `self.memory_enabled`: `:382-391, 415-416` — unroll, masks, loss accounting unchanged; twin sees the exact same segments and two-optimizer-update schedule). Twin configs: copies of stage1/cotrain yamls with `memory.enabled: false` (build path checkpoint-identical to legacy, `VLA_JEPA.py:122-129`), same `sample_mode: contiguous_segment` and all trainer knobs; stage1-twin warm-starts from the same `vlajepa_memv1_video` final; `seed: 42` + deterministic per-(epoch,index,seed) segment RNG (`datasets.py:1668-1683`) ⇒ identical data order ⇒ per-step loss deltas are paired. Chain via the pipeline pattern (`submit_vlajepa_memv1_pipeline.sh:169-187`). **Archive twin checkpoints at 20000 and 27752** (T1.4 and T1.3 matching points) in addition to 10K saves. **Staged commitment:** compare at 40K first (matched to current live checkpoint); extend to 100K only past G2. Eval: T0.1 protocol (twin has one mode) at 40K/(70K)/100K, libero_10 @30–50 trials.
- **Data:** identical all_robot + SSV2. **Controls:** this IS the control; memv1-prior arm from the same checkpoints shows prior ≠ twin (gap #2); identical LR groups/freeze lists minus absent memory groups; parameter counts reported (2.770B vs 2.776B); **seed-noise floor from T1.4 mandatory before any twin-vs-memv1 claim.** Strawmen avoided: no comparison to allv2 (single-step sampler + pause-filter confound); no claims from training loss alone.
- **Metrics:** primary — paired per-episode LIBERO success (McNemar + bootstrap CI, pooled over checkpoints), interpreted against the T1.4 seed SD; secondary — matched-step `loss/vla_action_loss` delta curve; `wm_loss`/jepa unchanged (sanity: memory cannot touch them).
- **Logging/viz:** standard TB/wandb; per-step delta arrays from joined event files; episodes.jsonl. Figures: paired loss-delta vs step with zero line and T1.4 seed-noise band; grouped success bars at matched steps with CIs. **Table 1 row "no-memory trained twin" + Fig. "Training attribution".**
- **Cost:** stage1-twin ~40 GPU-h; cotrain-twin to 40K ~270 GPU-h (→100K +~330); evals ~30 GPU-h.

### T1.2 — Static-prior control: writes disabled, learned initial slots only (branch-and-finetune from 40K)
- **Hypothesis:** memv1's advantage over the twin requires recurrent state accumulation, not 6.3M parameters injecting a learned constant bias. Writes-disabled lands between twin and full memv1 on libero_10; if it matches memv1, recurrence is useless and the module is a static adapter.
- **Method:** flag `trainer.memory_freeze_state`: zero the per-timestep `update_mask` in `forward_sequence` (mask built `VLA_JEPA.py:471-475`) so `write` never fires and `read` always returns `initial_slots` through trained fusion. Branch two 12K-step runs from step_40000: (a) flag on, (b) reference flag off (shared with T2.3/T2.4 as the common reference branch) — same seed ⇒ identical segment stream. **Branching caveat (per review):** T0.0 archives weights; unless the step_40000 trainer state (`optimizer.bin`) is also archived (now an explicit T0.0 ops item), branches are **warm-starts with fresh optimizer/scheduler, not resumes** — internally consistent because the paired reference branch shares the identical warm-start, and stated in the paper. Eval both at 12K, 4 suites; branch (a) additionally in live and prior serve modes (must be identical — consistency check that writes truly do nothing).
- **Data/controls:** paired reference branch isolates the flag; T1.1 twin = no-parameters floor → 3-way decomposition {no-memory / static-prior / recurrent}.
- **Metrics:** paired LIBERO success; matched-step loss; gate trajectory under freeze (does it decay when state is constant?).
- **Logging/viz:** post-T0.0 memory scalars; 3-bar decomposition per suite with CIs. **Fig. "What part of memory matters".**
- **Cost:** 2 × 12K @ ~2.1 s/step ≈ **110 GPU-h** + ~10 evals.

### T1.3 — Mechanism baseline: FIFO token buffer vs gated recurrence at matched parameters
- **Hypothesis:** the learned gated update outperforms a FIFO buffer of past projected action-marker summaries with identical read/fusion path. If FIFO matches, the contribution reduces to "give the policy any past tokens" — the standard reviewer kill-shot (Ni et al. 2022 / POPGym / TTM lineage). This is the in-model "trivial alternative" — explicitly NOT a retrieval system, consistent with the design philosophy; it is also the A7 fixed-window baseline the project's own ablation matrix demands (Evaluation-Plan.md:154-165).
- **Method:** consume `framework.memory.short_term.update: fifo` in `RecurrentMemory.write` (`recurrent_memory.py:231-279`): mean-pool `source_projection(source_norm(tokens))` per decision, shift into the 8 slots (drop oldest), skip gate/candidate nets; `read()` and fusion untouched (identical fusion capacity; report both param counts). Train with the exact stage1 recipe from the same memv1_video warm start, then a **cotrain pilot to ~28K, matched-step comparison at step 27752** — the gated arm's 20K checkpoint no longer exists (pruned); 27752 is the earliest surviving gated checkpoint (verified). Primary readout = the **T0.2 offline dose-response** (high power, cheap), secondary = paired libero_10 at matched 27752.
- **Data/controls:** same corpora; gated-vs-FIFO paired on identical data order (seed 42); bypass eval on both arms nets out shared fusion.
- **Metrics:** offline paired Δloss@J=8 gated vs FIFO; burn-in-curve shape (order-sensitivity: FIFO is order-preserving but ungated — the reversed-order control from T0.2b discriminates); libero_10 paired success at 27752.
- **Logging/viz:** same offline parquet + eval JSONL. **Table "Memory mechanism ablation at matched parameters"** {gated, FIFO, static-prior, twin} × {offline Δ, libero_10 success, params}.
- **Cost:** stage1 ~30 + ~28K cotrain ~190 + evals ~20 ≈ **240 GPU-h.** Descope option under budget pressure: stage1-only + offline battery (~35 GPU-h) with the closed-loop cell deferred.

### T1.4 — Seed-noise floor for the twin comparison *(promoted from Tier 2 per review; mandatory)*
- **Hypothesis:** the memory−twin effect exceeds 2× the seed-induced SD of full training runs. Without this, the entire attribution story rests on n=1 run per arm — the plan's own docs warn the 78/92/98/96 reference is "a one-seed reference, not a confidence interval".
- **Method:** 2×2 cell {memory-on, twin} × {seed 42, seed 1042} at matched step **27752** (the earliest step where the seed-42 memory arm has a surviving checkpoint): run seed-1042 memory-on and seed-1042 twin to ~28K, saving at 27752; seed-42 memory@27752 exists (archived); seed-42 twin@27752 archived by T1.1. Eval all four at libero_10 @30 trials under the T0.1 protocol. Loss-curve seed SD is additionally free from TB event files at every step (loss curves survived the checkpoint pruning).
- **Data/controls:** identical corpora and configs, seed only varies; per-step paired loss deltas within seed.
- **Metrics:** seed SD of (memory − twin) success delta; seed SD of matched-step loss delta; the T1.1 headline is reported as effect ± seed SD.
- **Logging/viz:** **Appendix "Noise budget" table** + shaded seed band on Fig. "Training attribution".
- **Cost:** 2 × ~28K cotrain ≈ **~380 GPU-h** + ~15 evals. Descope option: twin replicate only to 20K (twin@20K pairs archived) + memory replicate to 27752 (~330 GPU-h total); loss-based SD reported at all steps regardless.
